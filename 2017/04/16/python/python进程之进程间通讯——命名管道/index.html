python进程之进程间通讯——命名管道

1.简介

  • 命名管道：无名管道的一个扩展，无名管道是程序运行时存在，命名管道是持久的，一旦创建，所有有权限进程都可以访问
  • 命名管道是单向管道，只能以只读或只写方式打开，如果要实现双向通信，必须打开两个管道
  • 命名管道创建：os.mkfifo(path) ，读写和操作文件一样

  • 非阻塞：执行设备操作时，如果不能获取资源直接返回，可以使用轮询的方式进行设备操作
